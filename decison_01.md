3. 执行方式：怎么成交  
4. 出场规则：何时卖

这说明你的直觉已经是对的：T 日中午出信号以后，下午怎么进场，本身就是 alpha 的一部分。

我甚至会更直接一点说：

入场可以从哪几条路径优化

我建议你把“入场”拆成 3 层研究，不要混在一起。

1. 先优化“买谁”：做 veto filter，而不是只做排序
你现在已经能找到一些强票，但基金经理更在意的是：能不能先排掉明显会亏的票。

下一步建议不是继续堆更多评分项，而是研究“禁买条件”：

•  下午 13:30 仍然弱于下午开盘价，不买
•  T 日下午最大回撤超过阈值，不买
•  下午量能结构异常，不买
•  上午强，但下午承接弱，不买
•  题材/板块联动弱，不买

一句话：  
下一阶段先做 downside filter，比继续做 upside ranking 更值钱。



2. 再优化“何时买”：固定时点 vs 条件触发
你现在的 src/research_timing.py 属于“固定时点入场”研究，这是好起点，但还不够。

我建议你把入场实验分成两类：

A. 固定时点
•  13:00
•  13:05
•  13:15
•  13:30
•  13:45
•  14:00
•  14:30

这是 baseline。

B. 条件触发
比固定时点更重要。比如：

•  下午开盘后第一次回踩不破上午收盘价再买
•  13:30 前重新站上下午 VWAP 再买
•  下午前 30 分钟收益转正再买
•  下午量比低于某阈值且价格不弱再买
•  突破下午开盘后局部高点再买

这类规则更接近真实交易决策。

建议：固定时点只是对照组，条件触发才是主研究对象。



3. 最后优化“怎么买”：一次买满 vs 分批成交
这个层面很多人忽略，但对短线策略非常重要。

你可以研究：

•  13:00 一次性买入
•  13:00 先买 1/2，13:30 确认再补
•  下午首次回踩承接成功再补仓
•  只在价差不超过某阈值时成交

这个问题本质上是：执行方式是否能降低坏价格成交的概率。



我建议你怎么做“入场研究”的具体框架

第一类研究：T 日下午的路径分类
把下午走势先分型，而不是只比较平均值。

比如分成：

•  开盘即强，单边上行
•  开盘回落，后面收复
•  开盘冲高，随后走弱
•  高波动震荡
•  尾盘才启动

然后看每一类路径，对应的 T+1 表现是否不同。

这很关键，因为“最佳入场时间”往往不是全局唯一的，而是路径依赖的。



第二类研究：入场规则的联合收益
不要只看“买入到 T 日收盘”的收益，要看更完整的标签：

•  买入到 T 日收盘
•  买入到 T+1 开盘
•  买入到 T+1 10:00
•  买入到 T+1 最高点
•  买入到既定 exit 规则

否则你可能会选出一个“当天下午看着最好”，但对次日没有优势的买点。



第三类研究：入场和出场必须联合优化
这一点很重要。

最优 entry 不一定对应最优 exit。

你现在 src/research_timing.py 已经有一点这个方向了：在做买卖时点组合。但建议升级为真正的矩阵：

•  行：不同 entry 规则
•  列：不同 exit 规则
•  单元格：期望收益 / 胜率 / 最大回撤 / 样本数 / 稳定性

这样你找的是“最优组合”，不是局部最优。



用 Karpathy autoresearch 思想，我的建议是什么

我认为你现在的方法论方向是对的，但 src/autoresearch.py 目前更像：

特征扫描引擎  
而不是  
完整交易决策研究引擎

它现在主要在做：
•  提取特征
•  算相关性
•  五分位比较
•  记录结果

这对发现线索很有用，但不够支撑交易规则迭代。  
下一步建议把 autoresearch 从“找好特征”升级成“找好策略单元”。



我建议你把 autoresearch 升级成 3 层循环

第 1 层：Observation Loop
目的：发现线索

问题形式：
•  赢的票和亏的票，下午路径有什么差异？
•  哪些条件下 13:00 买入明显更差？
•  哪些信号只在特定市场环境下有效？

这一层重观察，不急着定规则。



第 2 层：Rule Loop
目的：把观察变成规则

问题形式：
•  如果 pm_am_vol_ratio < x 且 13:30 价格 > pm_open，是否提升次日期望？
•  如果 pm_max_drawdown > y，是否应禁止入场？
•  如果下午首 30 分钟走弱，延后到 14:00 是否更优？

这一层的关键是：  
一次只改一个变量，做可证伪的小实验。



第 3 层：Portfolio Loop
目的：看能不能变成可投策略

问题形式：
•  规则加入后，组合胜率/回撤/Sharpe 是否改善？
•  对滑点敏感吗？
•  只在一两天有效，还是在多周都稳定？
•  换手和容量是否还能接受？

这一层才是真正的基金经理视角。



Karpathy 风格下，我最建议你坚持的 6 条原则

1. 假设必须小而可证伪
不要做这种假设：

•  “加入更多下午因子会更好”

要做这种假设：

•  “对 score>=100 的股票，如果 13:30 仍低于下午开盘价，则禁买能降低最大回撤”

后者才是真正能被打脸、也能被保留的实验。



2. 每次实验只改一个决策点
比如只改：

•  entry time
•  entry filter
•  exit rule
•  universe
•  持仓数量

不要一轮实验同时改 4 件事，否则结果没法解释。



3. 记录失败实验，比记录成功实验更重要
很多策略研发最浪费时间的地方，是反复踩同一个坑。

建议你每个实验都记录：
•  假设
•  为什么觉得它成立
•  数据区间
•  改了什么
•  指标结果
•  为什么失败
•  下一步不再尝试什么

失败知识是最有复利的。



4. 评价指标要从“相关性”升级到“交易质量”
src/autoresearch.py 现在更偏相关性与五分位差异。  
下一步建议每个实验都强制输出：

•  平均收益
•  中位数收益
•  胜率
•  盈亏比
•  最大回撤
•  样本数
•  覆盖率
•  OOS 表现
•  滑点敏感性

否则会出现“统计上有点关系，但交易上没法赚”的问题。



5. 建立 champion / challenger 机制
不要每次都从零开始。

建议固定一个当前主策略 champion，然后每次新实验只和它比：

•  如果 challenger 在样本内赢，但 OOS 不赢，不升级
•  如果 challenger 收益高但回撤更差，也不升级
•  只有在多个维度都更优，才晋级

这样 research 会越来越稳。



6. 让 autoresearch 追求“信息增益”，不是只追求收益最大
有些实验即使没赚钱，也很有价值，因为它能明确告诉你：

•  某个入场条件其实没用
•  某个因子只在特定市况下有效
•  某个漂亮结果只是样本太小或幸存者偏差
•  某个规则一旦加上滑点和回撤约束就失效

这类“负结果”非常重要。  
Karpathy 式迭代的关键，不是每轮都赢，而是每轮都更少犯错。



我对你这个项目的一个核心建议
把 autoresearch 的研究对象，从“特征”升级为“决策单元”。

你现在的 src/autoresearch.py 更像：

•  特征提取器
•  因子扫描器
•  相关性评估器

而下一阶段更应该研究：

•  一个信号是否该交易
•  什么条件下允许入场
•  何时入场最优
•  配什么 exit 最优
•  是否值得占用组合仓位

也就是说，研究单元要从：

•  feature -> label

升级成：

•  decision_rule -> trade_outcome

这会让你的 autoresearch 更像真实投研，而不是单纯因子挖掘。



我建议你把下一阶段研究分成 4 个专题

专题 1：入场 veto 研究
目标不是“找到最强票”，而是“先排除最差票”。

专题 3：entry × exit 联合优化
这个很关键。

很多研究容易犯一个错误：  
先独立找最优 entry，再独立找最优 exit。

但真实情况常常是：

•  激进 entry 适合快 exit
•  保守 entry 适合趋势 exit
•  不同 entry，最优 exit 完全不同

所以你应该做的是组合矩阵：

•  行：entry 规则
•  列：exit 规则
•  指标：收益、胜率、回撤、盈亏比、样本数、稳定性

你现在 src/research_timing.py 已经有这个雏形，下一步建议把它变成主研究框架之一。

我很建议你尽快加这一层。

因为短线信号通常不是“始终有效”，而是：

•  在震荡市有效
•  在分歧转一致日有效
•  在指数走弱时失效

所以每个实验都建议增加“市场状态标签”：

•  大盘涨 / 跌 / 震荡
•  热门板块强 / 弱
•  连板情绪强 / 弱
•  当日涨停家数高 / 低
•  风格偏资源 / 科技 / 防御

这样你会更快发现：
不是策略不行，而是策略只适合某种 regime。

第三层：研究 entry × exit 组合
这是从“研究者思维”变成“策略管理者思维”的关键。

你不再问：
•  最佳买点是什么？
•  最佳卖点是什么？

而是问：
•  最佳交易组合是什么？

这是完全不同的层次。



第四层：研究 regime
如果不做这层，很多实验会反复给你错觉。

因为你可能会得到这样的结论：

•  规则 A 在某段样本很强
•  过几周突然失效
•  你以为是策略退化
•  其实只是市场状态变了

所以每个实验最好都加一列：
•  市场强弱
•  情绪强弱
•  板块扩散/聚焦
•  指数环境
•  连板高度/涨停家数等代理变量



用 Karpathy autoresearch 思维，我最推荐你加的 3 个机制

机制 1：实验必须有 baseline
任何新规则都必须回答：

•  它比当前默认规则好多少？
•  好在收益、胜率，还是好在回撤？
•  如果只能选一个提升，它提升的是哪一个？

没有 baseline，就很容易被“看起来不错”骗到。



机制 2：实验必须有 kill condition
每个实验开始前就写清楚：

•  什么结果算失败？
•  样本数低于多少不下结论？
•  OOS 不成立是否直接废弃？
•  加滑点后失效是否直接废弃？

这样 autoresearch 才不会无限发散。



机制 3：实验必须有升级条件
新规则不是“有点效果就留下”，而是要满足升级门槛。

比如：

•  OOS 收益不低于当前版本
•  最大回撤不恶化
•  样本覆盖率足够
•  逻辑可解释
•  执行复杂度没有明显上升

只有满足这些，challenger 才能升成 champion。



我建议你最近 2 周真正该做的，不是泛泛研究，而是这 4 个实验

实验 1：入场 veto
问题：
高分股票里，哪些下午行为一出现，就应直接放弃交易？

目标：
先减少亏损样本。



实验 2：固定时点 vs 条件确认 entry
问题：
13:00 买，和 13:30 条件确认后买，谁的收益/回撤比更优？

目标：
判断“快买”还是“确认后买”更适合你的策略。



实验 3：entry × exit 矩阵
问题：
不同买点和卖点的最优组合是什么？

目标：
找到真正可执行的交易闭环。



实验 4：regime filter
问题：
当前策略到底在哪些市场状态下有效？

目标：
决定策略何时开、何时关。



如果再压缩成一句最务实的话
你接下来最应该研究的是：

如何把“会选股”，升级成“会做交易”。

前一阶段你已经证明了自己开始有选股能力。  
下一阶段要证明的是：

•  你能不能少做错交易
•  能不能把买点买对
•  能不能把卖点卖对
•  能不能在不同市场里知道何时该出手、何时该收手

这才是从研究原型走向可投策略的分水岭。
